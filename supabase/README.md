# supabase setup

## migration order

Active CLI-managed migrations now live in `supabase/migrations/` and use the exact
timestamped versions recorded in the remote Supabase migration history.

Older local-only SQL drafts and superseded filenames were moved to
`supabase/migrations_archive/local_only_pre_remote_alignment/` so they remain
available for reference without causing `supabase` migration drift.

1. apply `migrations/20260413030025_make_app_alive.sql`
2. apply `migrations/20260413030101_profile_images.sql`
3. run `seeds/seed_z_code_taxonomy.sql`
4. ingest partner capability output generated by:
   - `npm run data:partner-capabilities`
   - file: `sample-data/ATLASDB-excel/partner-capabilities.seed.json`
5. apply `migrations/20260413030309_supervisor_navigator_competency.sql`
6. apply `migrations/20260413031148_atlas_app_runtime_cutover.sql`

If you need one of the archived historical drafts for reference, use the
archive path above rather than moving it back into `supabase/migrations/`.

## env vars for app runtime

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`

when these are missing, the app falls back to local seeds so development can continue without a live db.

## authorization foundation (roles, permissions, exceptions)

Migration `migrations/20260411041933_authorization_foundation.sql` adds:

- `atlas.permissions`
- `atlas.role_permissions`
- `atlas.user_permission_exceptions`
- `atlas.authorization_settings`

It also introduces helper functions for permission checks and Row-Level Security (RLS) policies on partner service-capacity tables. During rollout, legacy/public access remains controlled by settings rows:

- `allow_legacy_public_partner_capacity_read`
- `allow_legacy_public_partner_capacity_write`
- `allow_legacy_public_partner_capacity_delete`

Set these to `false` once auth-linked user role mapping and exception workflows are fully live.

## supervisor competency governance

Migration `migrations/20260413030309_supervisor_navigator_competency.sql` adds:

- `atlas.supervisor_navigator_assignments`
- `atlas.navigator_competency_assessments`
- `atlas.navigator_competency_assessment_answers`
- `atlas.v_supervisor_navigator_competency_rollup`

The rollup view computes supervisor-side navigator competency trend using weighted recent assessments:

- most recent average x `3`
- previous average x `2`
- third average x `1`

## profile images

- Supabase Storage bucket: `profile-images`
- Metadata table: `atlas.profile_images`
- App-facing resolved Uniform Resource Locator (URL) remains on `atlas.enrollees.avatar_url`
- The trigger in `20260413030101_profile_images.sql` keeps `avatar_url` synced to the current primary ready image for each enrollee.
