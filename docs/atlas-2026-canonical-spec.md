# ATLAS 2026 Canonical Technical Spec

This repository now treats the 2026 brief as the exclusive source of product behavior.

## Core Orientation

- Stabilization path: `Regulation -> Readiness -> Renewal`
- Product law: navigation under pressure
- Ethical posture: no person-level risk scoring
- Operational output: safest next action with explicit constraints

## Ontology Primitives

- `participant`: current phase state and readiness markers
- `pressureVector`: domain pressure + reversibility observations
- `capacityNode`: partner capability and available slots
- `routeCandidate`: dependency-aware sequenced intervention
- `memoryEvent`: verified milestone or blocker used in strip maps

## Pressure Domains

- Habitat stability
- Social networks
- Work and contribution
- Behavioral and physical regulation
- Mobility and access

## Hard Constraints

- No unsafe phase regressions
- Eligibility before route activation
- Block high-interference pathways
- Enforce dependency ordering
- Log explicit rationale for blocked options

## Route Scoring Factors

- Coverage weight
- Phase alignment weight
- Specialization match weight
- Reversibility weight
- Transfer cost penalty
- Interference penalty

## Collective Memory Evidence Rules

- Verification metadata required for each milestone
- Blockers categorized with unblock strategy
- Phase transitions require gate evidence
- Only verified events are shared memory-visible
- Corrections append; verified history is immutable
